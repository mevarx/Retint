# Export Format Specs

Each format draws from `{domain}.json` and must include three content blocks:

1. **Token block** — exact hex/px/ratio values (colors, type scale, radii, shadows, spacing unit, motion/easing)
2. **Taste directives** — each principle as an imperative: "When [Trigger], always [Decision] — because [Reason condensed to one clause]"
3. **Anti-patterns** — Restraint principles rewritten as "Never / Avoid" bullets

Anti-patterns carry as much value as positive specs — they prevent the AI from defaulting to template-looking output.
Anti-Slop rule applies here too: no "clean", "modern", "elegant". Cite specific px/hex, not vibes.

```markdown
globs: ["src/**/*.tsx", "src/**/*.css", "src/**/*.vue", "src/**/*.svelte", "app/**/*.tsx", "app/**/*.css"]
alwaysApply: false
---

## Design Tokens
[token block]

## Taste Directives
[directives]

## Anti-patterns
[anti-patterns]
```

`alwaysApply: false` + globs = auto-activates when those file types are in context. No manual `@rule-name` needed.

## Common Rules for All Exports

1. **Draw from `{domain}.json`** — never from raw step outputs or intermediate data.
2. **Anti-slop applies** — run the grep check against every export file before writing.
3. **Do not overwrite** — for targets that append to existing files (CLAUDE.md, copilot-instructions.md, etc.), always check if the file exists first and append, never replace.
4. **Create parent directories** — if `.cursor/rules/` or `.windsurf/rules/` etc. don't exist, create them.
5. **Domain in filename** — use `{domain}` in filenames (not `taste` or `design`) to prevent collisions across multiple extract runs.
